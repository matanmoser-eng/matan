## What I do differently

I use **Claude Code** as a core part of my PM workflow — not just for writing specs, but for building working prototypes directly in the product codebase.

The loop looks like this:

```
PRD → Claude Code (localhost + real data) → GitHub branch → staging canary → stakeholder review
```

This means when I hand something to engineering or design, it's not a wireframe — it's a running prototype they can click through, built against real data, on a real branch.
